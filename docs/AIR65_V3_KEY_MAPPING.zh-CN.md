# Air65 V3 按键映射

NuNuBar 0.13.0 在 macOS 上提供可点击的 Air65 V3 映射编辑器。映射由官方
Karabiner-Elements 执行，并限定为准确设备 `19F5:102B`，不会影响 Mac 内置键盘
或其他 NuPhy 型号。

## 依赖边界

- Air65 V3 状态灯不需要 Karabiner；
- 当前版本的按键和旋钮映射需要 macOS 官方 Karabiner-Elements，未安装或未运行
  时映射不会生效；
- NuNuBar 不内置键盘驱动，只会在用户确认配置变更后备份并合并自身管理的规则。

## 创建映射

1. 使用 USB 连接 Air65 V3，打开 NuNuBar > 键盘 > 按键映射。
2. 在键盘图上点选要修改的按键。点选旋钮时，再选择左旋、按下或右旋。
3. 从“系统动作”或“Codex”分组中选择目标动作。
4. 点击“创建”。确认框会显示 `~/.config/karabiner/karabiner.json` 和同目录
   时间戳备份的准确路径。
5. 确认后，绿色圆点表示映射已经保存。映射会替代该键原有功能。

## 旋钮

先在 NuPhyIO 将旋钮设置为：

- 左旋：`F21`
- 按下：`F22`
- 右旋：`F23`

这三个输入已经在 Air65 V3 上确认。NuNuBar 将它们视为三个互不覆盖的映射，
因此可以把左旋设为“Codex · 上一个任务”、按下设为“Codex · 新建任务”、右旋设为
“Codex · 下一个任务”。删除其中一个方向不会删除另外两个方向。

NuNuBar 0.13.1 build 48 已于 2026-07-19 完成端到端实机验收：实体右旋以
`F23` 进入 Karabiner，转换为 `Command-Shift-]`，Codex 随即切换到下一个任务。

输入标签旁的波形按钮会开启 8 秒原始载波检测。检测时应让 NuNuBar 位于前台。
如果 Codex 仍在前台，Codex 限定规则会先消费 `F23`，此时 EventViewer 显示的是
映射后的快捷键而不是原始 `F23`，属于正常结果。若显示 Fn/地球键，才说明事件
进入了另一条全局 `F24` 黄色键路线，不能作为旋钮验收结果。

## Codex 动作

可选动作包括新建任务、切换侧边栏、底部面板、文件树、Review 面板、终端、浏览器、
上一个/下一个任务、快捷键面板和设置。Codex 规则仅在 Codex 位于前台时触发；在
其他 App 中，旋钮或按键保留原始事件。系统动作是全局映射，会替代该键原有功能。

已存在的黄色 `PGDN` / `F24` / Fn 规则会被自动识别。`F24` 只是该历史成功路线
的输入事件，只在当前映射旁边作为技术状态显示。需要修改键盘自身输出时，可使用
编辑器中的“打开键盘层设置”按钮进入 NuPhyIO。

原生 Fn 暂不开放映射，因为它没有可供 Karabiner 重映射的普通输入事件。删除按钮只
删除当前实体键或旋钮方向的 NuNuBar 规则；其他 Karabiner profile 和规则保持不变。

本功能不修改 Codex Hooks、NuNuBar 状态灯配置或键盘固件，也不需要进入 DFU。
