# Air75 V3 按键映射

NuNuBar 0.15.0 为准确型号 Air75 V3 `19F5:1028` 增加独立的可点击 75% ANSI
配列，包含功能键行、右侧导航列和右上角旋钮，不复用 Air65 V3 的实体键位。

## 能力边界

- 状态灯同步不依赖 Karabiner；
- 按键和旋钮映射需要 macOS 官方 Karabiner-Elements；
- 每条规则都限定到准确设备 `19F5:1028`；
- 不覆盖 Air65 V3 规则、其他 Karabiner profile 或无关规则；
- Codex 动作只在 Codex App 位于前台时触发；
- Air75 V3 映射使用 USB 有线连接，不进入 DFU，也不需要自定义固件。

## 使用映射编辑器

1. 用 USB 连接 Air75 V3，打开 NuNuBar > 键盘 > 按键映射；
2. 点选实体按键或旋钮；
3. 选择系统动作或 Codex 动作；
4. 核对 Karabiner 配置路径和带时间戳的备份路径后确认；
5. 实测实体按键。删除操作只移除当前型号、当前控件的 NuNuBar 规则。

## 旋钮载体键

映射编辑器为旋钮保留三个独立输入：

| 实体动作 | NuPhyIO 载体键 |
| --- | --- |
| 左旋 | `F21` |
| 按下 | `F22` |
| 右旋 | `F23` |

先在 NuPhyIO 中把三种旋钮动作分别设为以上载体键，再在 NuNuBar 中创建映射。
波形按钮会实测当前方向是否真的发出对应功能键。三个方向通过检测并至少观察到
一次真实映射动作前，项目不得把 Air75 V3 旋钮路线标成已实机验证。

## 恢复

NuNuBar 每次修改前都会在 `~/.config/karabiner/karabiner.json` 同目录创建带时间戳
的备份。优先在 App 中删除单条映射；只有确认备份后没有其他 Karabiner 修改时，
才适合恢复整份备份。
