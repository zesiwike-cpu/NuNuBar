# Air75 V3 已验证有线路线

路线 ID：`air75-v3-macos-wired-1.0.14.6`

本路线只适用于 Apple Silicon macOS 上通过 USB 有线连接的准确 Air75 V3
`19F5:1028`。它使用 NuPhy 官方 HID `0001:0000`、64 字节控制接口，不使用
任何 V2 NuNuBar 固件，也不进入 V2 DFU 固件目录。

## 固件要求

NuNuBar 要求 Air75 V3 官方固件不低于 `1.0.14.6`。实机上的 `1.0.13.6`
虽然会回复已收到 `D6` 指令，却不会真正应用侧灯状态；NuPhyIO 官网也复现了
同样结果，因此当时的问题不在 Hooks 或 NuNuBar。先从 NuPhyIO 导出配置，再升级
至官方 `1.0.14.6` 后，官网恢复了配置，侧灯指令也立即可见。

NuNuBar 0.15.0 会读取官方固件版本，低于最低版本时阻止向导继续。用户需要单独
确认后在 `https://drive.nuphy.io` 完成官方升级；NuNuBar 不会静默更新或刷写
这款键盘。

## 抓包确认的协议

NuPhyIO 实际使用 report ID `0`、会话密钥 XOR 和带 ACK 的 64 字节报告。一次
侧灯状态由两条 `D6` 写入组成：

1. offset `9` 连续写入 8 字节：模式、亮度、速度、自定义 RGB 标志、颜色索引、
   红、绿、蓝；
2. offset `10` 单独写入 1 字节亮度。

实机确认官方常亮模式为 `2`、呼吸模式为 `3`，自定义 RGB 标志为 `0`。官方侧灯
没有独立闪烁模式，因此 NuNuBar 用每 500ms 一次、逐帧检查 ACK 的常亮开关实现。

## 待机始终常亮

即使 NuNuBar 当前状态是非黑色常亮，Air75 V3 固件仍可能在长时间无操作后关闭
全部灯光。这是键盘自己的自动休眠设置，不是 Codex 状态变化或 HID 发送失败。

[Air75 V3 官方手册](https://cdn.shopify.com/s/files/1/0268/7297/1373/files/Air75_V3_Quick_Guide_FAQ.pdf?v=1753438891)
说明：短按 `Fn + ]` 可以开启或关闭 Auto-sleep。需要状态灯始终可见时：

1. 在 NuNuBar 中把待机设为非黑色，并选择“常亮”；
2. 自动休眠当前开启时，短按一次 `Fn + ]`；
3. 停止操作并等待超过此前自动熄灯时间，确认待机侧灯仍然亮着；
4. 再触发真实 Codex 状态，确认能覆盖待机，并在结束后正常返回待机。

NuNuBar 不用模拟按键或高频 HID 轮询来对抗休眠；使用固件开关更确定，也不会
增加不必要的通信。

## 实机验收

- NuPhyIO 设置 99% 亮度的青色常亮后，左右侧灯都发生变化；
- 官网释放 HID 接口后，项目内 helper 依次显示了工作中、完成、等待和待机状态；
- NuNuBar 0.14.1 build 50 或更高版本能识别官方 `1.0.14.6`，持有长期 HID 会话，并使用
  系统已安装的 Codex Hook helper 路径。
- 用户已实际观察到真实 Codex 任务从橙色呼吸“工作中”切换为回复完成后的绿色
  常亮，证明系统安装的 `UserPromptSubmit` 与 `Stop` Hook 链路有效，而不只是
  直接灯光演示有效。
- NuNuBar 0.15.0 build 54 的每状态亮度已通过实机验收：用户把工作中亮度从
  `100%` 调至 `20%` 后观察到侧灯明显变暗；配置文件保存 `20`，发送日志记录
  `20%`，键盘 `D5` 状态回读的亮度字节为 `0x14`，三者完全一致。

`PermissionRequest` 已由同一 Hook 映射与自动化测试覆盖；普通用户在下一次真实
确认请求出现时仍应观察一次红色等待状态。

## 可复用的配置顺序

1. 核对键盘铭牌为 Air75 V3，USB 身份为 `19F5:1028`；
2. 切到有线模式，确认 HID `0001:0000` 和 64 字节报告；
3. 安装并启动 NuNuBar，授予输入监控权限；
4. 在 App 中读取固件。低于 `1.0.14.6` 时先导出 NuPhyIO 配置，取得明确确认后
   进行官方升级并重连；
5. 运行 App 灯光自检。成功后再配置并批准四个 Codex Hooks，NuPhyIO 不要与
   NuNuBar 同时打开；
6. 如需待机始终常亮，把待机设为非黑色常亮，并短按一次 `Fn + ]` 关闭自动休眠；
7. 验证真实 Codex 工作中、等待、完成、待机保持，以及 USB 拔插后的状态恢复。

可选的按键与旋钮映射属于独立 Karabiner 路线，请阅读
[`AIR75_V3_KEY_MAPPING.zh-CN.md`](AIR75_V3_KEY_MAPPING.zh-CN.md)。它不会改变
上述灯光验收，也不需要修改键盘固件。
