# NuNuBar 0.9.5

NuNuBar 0.9.5 is the first GitHub-ready release of the renamed app, guided
keyboard setup, configurable light states, wired Air65 V3 support, and the
cross-platform project structure.

Choose the route for the user's exact host and keyboard from
[`VERIFIED_PATHS.md`](../VERIFIED_PATHS.md). Air96 V2 ANSI on macOS is verified.
Air65 V3 on Apple Silicon macOS has verified transport, state, session, and
replug behavior with final dual-side visual sign-off pending. Other listed
profiles keep their published testing limitations.

## Downloads

- `NuNuBar-0.9.5-macOS-arm64.dmg` - macOS 14+ on Apple Silicon, when the
  release workflow has Apple signing and notarization credentials.
- `NuNuBar-0.9.5-macOS-arm64-UNNOTARIZED.dmg` - development fallback when
  those credentials are absent; review the Gatekeeper warning before use.
- `NuNuBar-Windows-0.9.5-x64.exe` and `.zip` - Windows 10/11 x64 USB Raw HID
  client. Windows does not yet support the Air65 V3 official control adapter.
- `SHA256SUMS.txt` - checksums generated from the exact release assets.

## Air65 V3 reliability fix

Air65 V3 keeps one device-global official control session. Earlier builds could
let a short-lived inspection helper negotiate a new session, leaving the menu
app with a stale key even though the keyboard still returned an acknowledgement.

Version 0.9.5 makes device inspection read-only and negotiates a fresh session
inside the shared cross-process transmission lock immediately before every
state update and host-rendered blink frame. It also coalesces unchanged and
in-flight state-file reconciliations instead of retransmitting them.

The exact `19F5:102B`, usage `0001:0000`, 64-byte Air65 V3 control interface
passed waiting-to-working transitions before and after `describe`, plus a real
USB unplug/replug cycle without restarting NuNuBar. Air65 V3 remains a no-flash
path and must never enter DFU for NuNuBar.

## Other changes

- Added configurable idle, working, waiting/error, and complete colors.
- Added solid, breathe, and blink effects plus configurable state durations.
- Added USB Raw HID protocol v3 and model-locked setup for Air60 V2 ANSI,
  Air75 V2 ANSI, Air96 V2 ANSI, and Halo75 V2 ANSI.
- Added safe Codex setup instructions, scoped Hook migration, macOS and Windows
  installers, CI, release packaging, and public-release documentation.
- Fixed Air65 V3 transport teardown from its own HID queue.

Air96 V2 ANSI and Air65 V3 have matching-hardware validation. Air60 V2 ANSI,
Air75 V2 ANSI, and Halo75 V2 ANSI USB firmware remain testing builds until each
exact retail model completes the published checklist.

### Air65 V3 稳定性修复

Air65 V3 的官方控制接口全局只保留一个会话。旧版本中，短生命周期的检测助手
可能重新协商会话，让菜单栏 App 留下过期密钥，因此即使收到 ACK，灯光也不会
实际改变。

0.9.5 将设备检测改为只读，并在每次状态更新和 App 生成的闪烁帧之前，于共享的
跨进程发送锁内重新建立有效会话。两秒状态文件兜底检查也会合并未变化或正在发送
的状态，不再重复传输。

准确的 `19F5:102B`、usage `0001:0000`、64 字节 Air65 V3 控制接口已经通过
检测前后的红色等待与橙色工作切换，并完成真实 USB 拔插恢复。Air65 V3 始终使用
官方固件，不进入 DFU，也不刷写 NuNuBar 固件。
