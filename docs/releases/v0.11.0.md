# NuNuBar 0.11.0

NuNuBar 0.11.0 makes keyboard recognition lead to the shortest safe path:
reuse a working keyboard setup, and prepare firmware only when the existing
status-light channel does not respond.

## New-keyboard flow

1. NuNuBar identifies the exact supported keyboard over wired USB.
2. Air65 V3 continues directly with its official no-flash control interface.
3. A V2 keyboard runs the in-app orange/green/red light self-test before any
   firmware preparation.
4. The user records the physical result. Visible changes preserve the installed
   firmware and continue directly to Codex. No change leads to USB and
   permission troubleshooting, then the existing model-locked firmware gates.
5. Codex Hooks remain a separate, explicit local configuration step. Hook
   trust is never approved automatically. The App shows the two affected
   configuration files, the pending-Hook approval step, and the return to
   NuNuBar for a Connected check.

Air65 V3 users also get an optional Yellow Key Shortcut section. It opens
with a clickable Air65 V3 layout and separates the physical yellow `PGDN` key,
`F24` carrier, and final Fn/Globe action. It opens NuPhyIO for the keyboard-side
assignment, warns that `F24` must not be duplicated on another physical key,
detects official Karabiner-Elements, and installs or removes only a
device-scoped rule after showing the exact config and backup paths. The full
path was physically verified with Karabiner-Elements 16.1.0 on macOS 15.1.

The App cannot infer a V2 firmware version from VID/PID. This functional check
therefore avoids unnecessary repeat flashes while keeping the exact-model,
backup, recovery-image, DFU, hash, and final-write confirmations for keyboards
that genuinely need NuNuBar firmware.

## Support status

- Air65 V3 on Apple Silicon macOS: official wired control, no flash.
- Air96 V2 ANSI on Apple Silicon macOS: hardware-verified v7 route.
- Air60 V2 ANSI, Air75 V2 ANSI, and Halo75 V2 ANSI: testing firmware.
- Windows: V2 USB client software-tested; matching hardware acceptance pending.

## 中文说明

NuNuBar 0.11.0 让“识别键盘”之后优先进入最短安全路线：现有灯光通道能用就直接
保留，只有不能响应时才准备固件。

### 新键盘流程

1. NuNuBar 通过有线 USB 识别准确的受支持键盘；
2. Air65 V3 继续使用官方免刷控制接口；
3. V2 键盘在任何固件准备前，先运行 App 内橙、绿、红三色自检；
4. 用户记录实体观察结果：灯光有变化就保留现有固件并直接接入 Codex；没有变化
   则先排查 USB 和权限，再进入原有的型号锁定固件流程；
5. Codex Hooks 仍是单独确认的本地配置，App 不会自动批准 Hook 信任。

Air65 V3 用户还会看到可选的“黄色键快捷功能”：App 可打开 NuPhyIO 完成实体
键盘图中点选黄色 `PGDN`，分别查看实体键位、`F24` 传递信号和 Fn/地球键动作。
App 可打开 NuPhyIO 完成键盘侧设置，并提示检查其他键没有重复使用 `F24`；随后
检测官方 Karabiner-Elements，在展示准确配置和备份路径后，只安装或移除设备限定
规则。完整链路已在 macOS 15.1、Karabiner-Elements 16.1.0 上通过实机验证。

向导会明确显示 `~/.codex/hooks.json`、`~/.codex/config.toml`、四个待批准事件，
以及批准后回到 NuNuBar 点击“重新检测”的步骤。用户不需要在对话中输入
`/hooks`。

App 无法仅凭 VID/PID 判断 V2 的固件版本，因此通过功能自检避免无意义的重复
刷写；真正需要固件时，准确型号、备份、恢复镜像、DFU、哈希与最终写入确认均
继续保留。
