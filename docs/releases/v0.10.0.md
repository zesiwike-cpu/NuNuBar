# NuNuBar 0.10.0

NuNuBar 0.10.0 turns the verified Air65 V3 USB result into a reusable App-owned
setup. It keeps all existing V2 model profiles and firmware artifacts.

## Codex handoff and release safety

- `START_HERE.md` provides one bilingual prompt for a user who downloads the
  repository and opens it in Codex.
- `script/preflight.py --json` performs read-only repository, host, App/Hooks,
  USB, and HID discovery before Codex chooses a route. It cannot install,
  configure, enter DFU, or flash.
- For V2 keyboards, the report deliberately keeps firmware compatibility
  unknown until the App self-test proves status lighting. A matching VID/PID
  never causes a repeat flash.
- The public macOS Release workflow now fails unless it can produce a Developer
  ID signed and Apple-notarized DMG. Ad-hoc builds remain development-only CI
  artifacts.

## Air65 V3

- Uses the exact official USB control interface `19F5:102B`, usage `0001:0000`,
  with 64-byte reports. No DFU or keyboard firmware flash is involved.
- Negotiates the official session, sends encrypted `GetBase` (`A0`), and waits
  for the keyboard's active light profile before reporting ready.
- Sends side-light data to offset `9` with the discovered profile in byte `7`.
  A valid ACK alone is not treated as visible-light proof.
- Adds an in-app orange/green/red self-test and restores the current Agent state
  after the test.
- Makes `agent-light describe` enumeration-only so it cannot replace the menu
  App's persistent Air65 V3 session.

Matching-hardware acceptance on macOS confirmed profile `0` discovery and the
visible orange, green, and red self-test sequence. Real Codex working and
complete transitions were already physically verified. Idle, a non-default
custom color, and unplug/replug on 0.10.0 remain explicit follow-up checks.

## V2 support retained

| Model | USB VID:PID | Status |
| --- | --- | --- |
| Air60 V2 ANSI | `19F5:3255` | Testing firmware |
| Air75 V2 ANSI | `19F5:3246` | Testing firmware |
| Air96 V2 ANSI | `19F5:3266` | Hardware verified |
| Halo75 V2 ANSI | `19F5:32F5` | Testing firmware |

The four model-specific firmware files, manifests, SHA-256 checks, light maps,
and separate DFU confirmation gates are unchanged. Firmware must never be
interchanged between models.

- DMG: `NuNuBar-0.10.0-macOS-arm64.dmg`
- Local ad-hoc DMG SHA-256:
  `68402c330559b2e5abcfff81b244b7b4e869c7e074ceb8e37927611563bec9de`

This hash identifies the locally accepted unnotarized artifact only. A future
Developer ID signed and notarized public build will have a different hash and
must publish that exact value in its release checksums.

## 中文说明

NuNuBar 0.10.0 把 Air65 V3 已经成功的 USB 路线固化为可复用的 App 流程，同时
完整保留四款 V2 的型号配置和固件产物。

### Codex 交接与发布安全

- `START_HERE.md` 为下载仓库后交给 Codex 的新用户提供唯一的中英双语提示词；
- `script/preflight.py --json` 在 Codex 选择路线前，只读检查仓库、电脑、
  App/Hooks、USB 与 HID，不会安装、配置、进入 DFU 或刷写；
- V2 键盘的固件兼容性必须由 App 灯光自检证明，匹配 VID/PID 绝不会直接触发重刷；
- 面向普通用户的 macOS Release 缺少 Developer ID 签名或 Apple 公证凭据时会
  直接失败，临时签名包只保留为开发 CI 产物。

### Air65 V3

- 严格匹配官方 USB 接口 `19F5:102B`、usage `0001:0000` 和 64 字节报告，
  不进入 DFU，也不刷键盘固件；
- 握手后发送加密 `GetBase`（`A0`），读到键盘当前活动灯光配置后才进入就绪；
- 使用侧灯 offset `9`，并把自动读取的活动配置写入 byte `7`；收到 ACK 不再被
  当作肉眼灯光已经变化的充分证据；
- App 内加入橙、绿、红三色自检，结束后恢复真实 Agent 状态；
- `agent-light describe` 只枚举设备，不再替换菜单栏 App 持有的 Air65 V3 会话。

同一台零售 Air65 V3 已实机确认自动读出配置 `0`，并目视通过橙、绿、红自检。
真实 Codex 工作中和完成切换此前也已实机确认。待机、自定义非默认颜色和
0.10.0 拔插恢复仍作为后续明确验收项。

### V2 支持保留

Air60 V2 ANSI、Air75 V2 ANSI、Air96 V2 ANSI、Halo75 V2 ANSI 的独立固件、
清单、哈希、灯位映射和 DFU 二次确认流程全部保留。Air96 V2 已实机验证；另外
三款仍是测试固件，不能互刷，也不能因为 Air65 V3 成功而提升验证等级。
