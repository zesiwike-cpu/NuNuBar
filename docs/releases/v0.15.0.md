# NuNuBar 0.15.0

## Air75 V3 key mapping

- Added a clickable 75% ANSI layout for the exact Air75 V3 `19F5:1028`.
- Added ordinary keys, the function row, the navigation column, and separate
  left, press, and right knob controls.
- Generated Karabiner rules are restricted to the exact Air75 V3 device and do
  not replace Air65 V3 mappings or unrelated Karabiner configuration.
- Codex actions remain restricted to the frontmost Codex App.
- The knob uses `F21` for left, `F22` for press, and `F23` for right. Use the
  built-in input probe to verify these carrier events after assigning them in
  NuPhyIO.
- The keyboard-level settings button opens the current official NuPhyIO 2.0
  entry with the matching App language instead of the obsolete deep route.

Lighting does not require Karabiner. Air75 V3 status lights continue to use the
verified official wired interface and official firmware `1.0.14.6` or later.
For always-on idle lighting, the reusable setup uses the official short
`Fn + ]` Auto-sleep toggle instead of periodic HID polling.
Color-picker previews are debounced for 150 ms, preventing rapid pointer
updates from flooding the Air V3 HID interface and repeatedly restarting its
light effect. The settled color still previews on the keyboard for 3 seconds.
Each status now also stores an independent 0-100% brightness. Air75 V3 writes
that percentage to the official hardware brightness field, while Air65 V3 maps
it to the model's 0-24 range. Existing V2 firmware remains compatible and
receives brightness-adjusted RGB values without a protocol change. Older
palette files migrate to 100% without losing their colors or effects.

## Air75 V3 按键映射

- 为准确型号 Air75 V3 `19F5:1028` 增加可点击的 75% ANSI 配列；
- 支持普通按键、功能键行、右侧导航列，以及旋钮左旋、按下、右旋三个独立动作；
- Karabiner 规则只作用于准确的 Air75 V3，不会覆盖 Air65 V3 映射或其他
  Karabiner 配置；
- Codex 动作仍然只在 Codex 位于前台时触发；
- 旋钮使用 `F21` 左旋、`F22` 按下、`F23` 右旋。先在 NuPhyIO 分配载体键，
  再用 App 内输入检测完成实机确认。
- 键盘层设置按钮会按 App 语言打开当前 NuPhyIO 2.0 官方入口，不再使用旧的
  深层路由。

灯光同步不依赖 Karabiner。Air75 V3 状态灯继续使用已经验证的官方有线接口，
并要求官方固件 `1.0.14.6` 或更高版本。
需要待机始终常亮时，配置流程使用官方短按 `Fn + ]` 自动休眠开关，不进行周期性
HID 轮询。
颜色选择器会等待停止拖动 150 毫秒后再向键盘发送最终颜色，避免大量中间色指令
反复重启 Air V3 灯效；最终颜色仍会在键盘上预览 3 秒。
四种状态现在分别保存 `0–100%` 亮度。Air75 V3 会把百分比写入官方硬件亮度字段，
Air65 V3 则换算到该型号的 `0–24` 档位；现有 V2 固件不改变协议，通过缩放 RGB
实现亮度效果。旧配置会自动补为 `100%`，原有颜色和灯效不会丢失。
