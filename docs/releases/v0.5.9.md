# NuphyBar 0.5.9

This release makes NuphyBar event-driven and self-healing while keeping the same UI and keyboard animations.

## Downloads

- `NuphyBar-0.5.9-macOS-arm64.dmg` — macOS 14+ on Apple Silicon
- `NuphyBar-Air60-V2-stable-v7.bin` — **NuPhy Air60 V2 ANSI only; unchanged from 0.5.8**
- `SHA256SUMS` — checksums for the release assets

## What's new

- Added Antigravity lifecycle integration using Google's official plugin Hooks.
- Agent lifecycle changes now wake NuphyBar through a macOS system notification instead of being discovered by one-second polling.
- A persistent, non-exclusive HID manager listens for keyboard connection and removal events instead of repeatedly opening the keyboard.
- Failed reports discard the stale HID session and retry with bounded backoff.
- Mac wake proactively rebuilds the HID session and replays the latest Agent state after recovery.
- Complete and error indications return to idle after about 15 seconds using exact expiration timers.
- Release binaries strip local symbols, and firmware candidate verification remains active under `python -O`.

The state file remains the durable local source of truth. Notifications are only a wake-up signal, and a five-second fallback poll is used if notification registration fails. Bluetooth still receives only one two-byte state report when the displayed state changes; every animation frame remains on the keyboard.

## Installation notes

The DMG targets macOS 14+ on Apple Silicon. It is ad-hoc signed and has not been notarized with an Apple Developer ID. If macOS blocks the first launch, right-click the app, select **Open**, and allow Input Monitoring in System Settings.

The included `stable-v7` firmware binary is unchanged. Existing 0.5.8 users who already flashed `stable-v7` only need to update the macOS app and must not reflash the keyboard.

Before flashing for the first time, confirm the exact keyboard model, export the VIA layout, keep the official recovery image, and verify SHA-256 checksums. Never flash the Air60 V2 firmware to another Air V2 size.

## 中文说明

这一版在保持 UI 和键盘灯效不变的前提下，把 NuphyBar 改成了事件驱动，并加入完整的 HID 自动恢复机制。

### 下载

- `NuphyBar-0.5.9-macOS-arm64.dmg` — macOS 14+，Apple Silicon
- `NuphyBar-Air60-V2-stable-v7.bin` — **仅限 NuPhy Air60 V2 ANSI；与 0.5.8 完全相同**
- `SHA256SUMS` — Release 文件校验值

### 本次更新

- 使用 Google 官方插件 Hooks 接入 Antigravity 生命周期状态；
- Agent 状态变化通过 macOS 系统通知立即唤醒 NuphyBar，不再每秒轮询；
- 使用长期存在、非独占的 HID 管理器监听键盘连接与移除，不再反复打开键盘；
- 报告发送失败后丢弃失效会话，并按受控间隔自动重试；
- Mac 唤醒后主动重建 HID 会话，恢复后自动补发最新 Agent 状态；
- 完成和错误提示使用准确的过期时间，约 15 秒后恢复空闲；
- Release 二进制剥离本地符号，固件候选校验在 `python -O` 下也不会失效。

状态文件仍然是可靠的本地状态存档，系统通知只负责“叫醒”应用；如果通知注册失败，会自动启用五秒一次的兜底检查。蓝牙仍然只在显示状态变化时收到一个两字节报告，动画的每一帧仍由键盘本地生成。

### 安装提示

DMG 使用 ad-hoc 签名，尚未经过 Apple Developer ID 公证。首次启动如被 macOS 拦截，请右键应用选择“打开”，并在系统设置中允许输入监控。

本次附带的 `stable-v7` 固件没有变化。已经在 0.5.8 刷过 `stable-v7` 的用户只需要更新 macOS App，**不需要重新刷键盘**。

第一次刷写前必须确认型号、备份 VIA、准备官方恢复文件并核对 SHA-256。Air60 V2 固件不能刷入其他 Air V2 尺寸。
