# NuNuBar 0.13.1

NuNuBar 0.13.1 restores Air65 V3 status lighting automatically after NuPhyIO
or another HID client replaces the keyboard's device-global control session.

## Changes

- Codex `UserPromptSubmit` and `PermissionRequest` Hooks now persist a delivery
  checkpoint in addition to the visible state.
- A new checkpoint resends the active color even when the state remains
  `working` or `waiting`.
- If that send detects a stale Air65 V3 session, NuNuBar automatically performs
  the existing reconnect, handshake, and latest-state replay sequence.
- Repeated `PostToolUse` events remain coalesced to avoid unnecessary traffic.
- The Air65 mapping editor now includes a short input probe that reports the
  actual F-key carrier or Fn/Globe result for physical acceptance testing.

## Compatibility

- Air65 V3 continues to use its verified official wired HID interface without
  a firmware flash.
- Air65 V3 knob mappings remain left `F21`, press `F22`, and right `F23`.
- The yellow `PGDN` key may remain on its distinct `F24` to Fn/Globe mapping.
- Status lighting does not require Karabiner. Air65 V3 key and knob mappings in
  this release require official Karabiner-Elements on macOS.
- Air60 V2, Air75 V2, Air96 V2, and Halo75 V2 keep their existing firmware and
  validation status.

## Hardware acceptance

NuNuBar 0.13.1 build 48 passed acceptance on a retail Air65 V3 connected by USB
to macOS 15.1 arm64. A real Codex prompt restored the configured working light.
With the knob configured in NuPhyIO as left `F21`, press `F22`, and right `F23`,
a physical right rotation produced `Command-Shift-]` and switched Codex to the
next task. No keyboard firmware was flashed.

NuNuBar 0.13.1 会在 NuPhyIO 或其他程序改写 Air65 V3 的设备级 HID 会话后自动
恢复状态灯。每次 Codex 提交提示词或请求确认时，Hook 都会生成一次新的下发检查点；
即使当前仍是“工作中”或“需要确认”，App 也会重新发送当前颜色。若旧会话已经失效，
NuNuBar 会自动重新握手并恢复最新状态。

普通工具调用仍会合并，避免每次 `PostToolUse` 都重复写入。Air65 V3 的
`F21`/`F22`/`F23` 旋钮映射、黄色键 `F24` 地球键映射，以及四款 V2 键盘路线
均保持不变。映射编辑器新增短时输入检测，可直接判断旋钮是否真的输出了预期载波。
状态灯不需要 Karabiner；本版本 Air65 V3 的按键和旋钮映射需要 macOS 官方
Karabiner-Elements。

0.13.1 build 48 已在 macOS 15.1 arm64 与零售 Air65 V3 有线连接上完成实机
验收：真实 Codex 提示词恢复了用户设置的工作灯；NuPhyIO 设为左旋 `F21`、按下

任务。整个过程没有刷写键盘固件。
